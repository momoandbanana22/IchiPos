# CLAUDE.md

このリポジトリで作業する AI エージェント（Claude Code）向けの指針。Claude Code はこのファイルを毎回自動で読み込む。

## 開発プロセスは DEVELOPMENT.md に従う

TDD・ドキュメント駆動・SSoT・「作業中に見つけたことの扱い（気づきは Issue に起票する）」・Git / ブランチ運用・リリース手順は、すべて [DEVELOPMENT.md](./DEVELOPMENT.md) が正典。**実装を始める前に必ず読むこと。** ここに内容を複製しない（SSoT）。

関連する設計書（`doc/01〜04`）・`README.md`・`config/config.yaml.example` の扱いも DEVELOPMENT.md に従う。

## AI エージェントの行動規範

上記の開発プロセスに加えて、エージェントとして以下を守る。

- **不可逆・外向きの操作を無断で行わない。** ブランチ・タグの削除、force push、ファイル削除、外部への送信などは、頼まれていない限り実行せず、必要なら先に確認する。
- **自分の既定より既存の慣習を優先する。** 何かする前に `git log`・既存ブランチ・過去 PR など、リポジトリの実際の状態から慣習を読み取り、それに合わせる。既定と食い違うときは慣習を優先する。
- **実行していない作業を「やった」と報告しない。** 起票・修正・テストなど、やっていないことを完了として報告しない。テストが落ちたら落ちたと、スキップしたらスキップしたと、そのまま伝える。
- **コスト意識を持つ。** CI やバックグラウンド処理を張り付いて監視（ポーリング）しない。完了通知を待つか、必要になったときに確認する。
