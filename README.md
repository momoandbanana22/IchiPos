# IchiPos

IchiPos は、Windows 11 上で動作するアプリケーションです。GUI（画面操作）と CLI（コマンドライン）の両方で使えます。

ひとつの投稿内容（本文と画像）を、
1. Misskey に投稿し、
2. X の投稿画面をブラウザで開きます。

## 機能

- 投稿テキストを文字列または `.txt` ファイルで指定できる（GUIでは画面から直接入力、またはファイル読み込み）
- 投稿テキスト中の `{date}` を実行時の日付（`YYYY/MM/DD`）に置換する
- 投稿画像（添付画像）をフォルダで指定できる
- Misskey に投稿する（画像添付対応）
- X の投稿画面をブラウザで開く
- 画像がある場合、1枚目をクリップボードにコピーする（X への貼り付け用）
- 投稿後に添付画像を削除する（GUIでは事前チェックボックス、CLIでは投稿後の確認）

## 動作環境

- OS：Windows 11
- 言語：C#
- フレームワーク：.NET 10
- アプリ形式：実行ファイル1つで GUI（WPF）・CLI（コンソール）の両方に対応

## GUIモードとCLIモードの切り替え

`IchiPos.exe` は、起動時のコマンドライン引数の有無で自動的に動作モードを切り替えます。

| 起動方法 | モード |
|---|---|
| 引数なしで起動（エクスプローラーからのダブルクリックを含む） | GUIモード |
| 引数ありで起動（`IchiPos.exe <content> ...` など） | CLIモード |

## ダウンロード

[Releases](https://github.com/momoandbanana22/IchiPos/releases) から最新の `IchiPos-vX.X.X.zip` をダウンロードして展開してください。

展開後の構成：

```text
IchiPos.exe
config\
    config.yaml.example
```

### 前提条件

.NET 10 ランタイムが必要です。Microsoft の公式サイトから **.NET 10** をインストールしてください。

インストール確認：

```text
dotnet --version
```

`10.x.x` と表示されれば準備完了です。

## 設定ファイル

設定ファイルは YAML 形式で、実行ファイルと同じディレクトリの `./config/config.yaml` に配置します。

同梱の `config\config.yaml.example` をコピーして作成してください。

```text
copy config\config.yaml.example config\config.yaml
```

各フィールドの説明は [config/config.yaml.example](config/config.yaml.example) を参照してください。

設定ファイルが存在しない場合、または必須フィールドが未設定の場合はエラーとなります。

### Misskey アクセストークンの取得

1. Misskey インスタンスにログインする
2. 「設定」→「API」を開く
3. 「アクセストークンの発行」から新しいトークンを発行する
4. 必要な権限：**ノートを作成する**、**ドライブを操作する**
5. 発行されたトークンを `config.yaml` の `access_token` に設定する

## GUIモード

引数なしで `IchiPos.exe` を起動すると、画面から操作できます。

1. **投稿内容**：テキストボックスに直接入力するか、「ファイルから読み込む...」で `.txt` ファイルを読み込みます（読み込み後も編集できます）
2. **画像フォルダ**：「参照...」で添付画像フォルダを選択します（「クリア」で選択解除）
3. **投稿後に画像を削除する**：チェックを入れておくと、投稿完了後に添付画像を自動的に削除します（画像フォルダ未選択の場合はこのチェックボックスは操作できません）
4. **投稿する**：押すと Misskey への投稿・X 投稿画面の起動までを実行します。実行中はボタン等が無効化され、二重投稿を防ぎます
5. 結果は画面下部の結果ログに時系列で表示されます（成功は緑、エラーは赤）

`{date}` の日付埋め込みは CLI と同じ規則で、投稿実行時に反映されます。

ウィンドウは投稿後も閉じないため、内容を編集して続けて投稿できます。

### クリップボード画像の貼り付け

画像をコピーした状態で、ウィンドウ内のどこにフォーカスがあっても **Ctrl+V** を押すと、その画像を添付画像として貼り付けられます（画像フォルダ欄に、貼り付けた画像を保存した一時フォルダのパスが表示されます）。

- クリップボードに画像がない場合（文字列のみ等）は、通常の貼り付け動作（投稿内容欄への文字列貼り付け等）を妨げません。
- **1回の貼り付けにつき1枚のみ対応**です。貼り付けるたびに前回貼り付けた画像は置き換わります（3回貼り付けても3枚にはなりません）。複数枚まとめて投稿したい場合は、従来どおり「画像フォルダ」で複数枚入ったフォルダを指定してください。
- エクスプローラーで複数の画像ファイルを選択してコピーする操作には対応していません（貼り付けても反応しません）。単一の画像（ブラウザの「画像をコピー」、Snipping Tool 等）のみ対応です。
- 画像フォルダの「参照...」による選択・「クリア」とは互いに排他です。貼り付け後にフォルダを選択し直すと、貼り付けた画像は破棄されます。

## CLIモード

引数を付けて `IchiPos.exe` を起動すると、従来どおりコマンドラインで操作できます。

```text
IchiPos.exe <content> [--image-path <folder>]
IchiPos.exe --version
```

| 引数 | 必須 | 内容 |
|---|---|---|
| `<content>` | 必須 | 投稿テキスト、または `.txt` ファイルパス |
| `--image-path <folder>` | 任意 | 添付画像を格納したフォルダパス |
| `--version` | 任意 | バージョンを表示して終了する |

### 実行例

```text
# テキストをそのまま投稿
IchiPos.exe "今日のランチ"

# テキストファイルから投稿
IchiPos.exe post.txt

# 画像フォルダを指定して投稿
IchiPos.exe post.txt --image-path C:\Pictures\today

# バージョン確認
IchiPos.exe --version
```

### 投稿テキストへの日付埋め込み

投稿テキスト中に `{date}` が含まれる場合、実行時点の日付（`YYYY/MM/DD`）に置換されます。

```text
IchiPos.exe "今日は{date}です"
# → 今日は2026/07/03です
```

- コマンドラインの文字列、`.txt` ファイルのどちらから読み込んだテキストでも置換されます。
- `{date}` は大文字・小文字を区別します（`{Date}` や `{DATE}` は置換されません）。
- 複数含まれる場合はすべて置換されます。

### 画像付き投稿について

`--image-path` を指定すると以下の動作になります。

- **Misskey**：フォルダ内の画像をすべて添付して投稿します。
- **X**：X の Intent URL では画像を渡せないため、X 投稿画面を開いた後、  
  **1枚目の画像を自動的にクリップボードにコピー**します。  
  X 下書き画面で **Ctrl+V** を押すと画像を貼り付けられます。

> 画像が複数枚ある場合、2枚目以降は手動で X に添付してください。

### 投稿後の画像削除（CLIモード）

投稿が完了すると、添付画像がある場合にユーザーへ確認したうえで画像を削除します。

```text
画像を削除してよいですか？（N枚）(y/n):
```

| 入力 | 動作 |
|---|---|
| `y` または `Y` | フォルダ内の添付画像をすべて削除する |
| それ以外 | 削除しない |

GUIモードでは、投稿後の確認ダイアログではなく、投稿前の「投稿後に画像を削除する」チェックボックスで事前に設定します。

## 開発者向け: ソースからビルド

```text
git clone https://github.com/momoandbanana22/IchiPos.git
cd IchiPos
dotnet build -c Release
```

ビルド後の実行ファイル：

```text
IchiPos.Core\bin\Release\net10.0-windows\IchiPos.exe
```

実行時は `config/` ディレクトリを `IchiPos.exe` と同じ場所に置いてください。
